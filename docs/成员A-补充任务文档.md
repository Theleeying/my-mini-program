## 补充任务清单

### 任务 A1：收藏同步支持失物招领 🎯

**现状**：`pages/user/my-favorites/` 只收藏和展示二手商品（`goods` 集合）。

**目标**：收藏同时覆盖二手商品和失物招领信息。在 `favorites` 集合中新增 `lostId` 字段，我的收藏页支持双 tab 展示。

**改动范围**：仅 `pages/user/my-favorites/` 和 `pages/index/`（热门推荐添加收藏入口）。

**不影响**：成员C（失物招领模块只负责发布/浏览，不碰收藏）。

**具体点**：

| # | 内容 |
|---|------|
| 1 | `favorites` 集合字段扩展：`goodsId` -> `itemId` + `itemType`（`goods` / `lost_found`） |
| 2 | 首页热门推荐每条右侧加 ⭐ 收藏按钮 |
| 3 | 我的收藏页改为商品/失物双 tab 切换 |
| 4 | 收藏/取消收藏逻辑适配两种类型 |

### 任务 A2：公告详情页 📢

**现状**：首页轮播图只展示标题，点击无反应。公告数据是成员A管理的 `announcements` 集合。

**目标**：新增公告详情页，首页轮播图和列表中的公告可点击进入查看全文。

**改动范围**：新增 `pages/common/announcement-detail/` 页面，修改首页 WXML 绑定点击事件。

**不影响**：任何人。公告数据归属成员A。

**具体点**：

| # | 内容 |
|---|------|
| 1 | 新建 `pages/common/announcement-detail/` 页面（4个文件） |
| 2 | `app.json` 注册路由 |
| 3 | 首页轮播图每个 item 绑定 `bindtap` → `navigateTo` 公告详情 |
| 4 | 公告详情页展示：标题 + 封面图 + 正文 + 发布时间 |

### 任务 A3：全局搜索中间页 🔍

**现状**：首页搜索只跳到二手交易 TabBar 页（`switchTab`），体验割裂且只搜二手。

**目标**：新增一个独立的搜索结果页，同时搜索 `goods` 和 `lost_found`，按分类/类型双维度过滤。

**改动范围**：新增 `pages/common/search-results/` 页面，修改首页搜索逻辑。

**不影响**：B/C 各自模块内部搜索逻辑仍由他们自己实现。全局搜索只是搜完跳转到他们的详情页。

**具体点**：

| # | 内容 |
|---|------|
| 1 | 新建 `pages/common/search-results/` 页面 |
| 2 | 同时查询 `goods` + `lost_found`，按关键词模糊匹配 |
| 3 | 结果分 tab（全部/二手/失物）展示 |
| 4 | 点击结果跳转到对应详情页（B/C 负责） |
| 5 | 搜索结果高亮关键词 |

**不影响**：任何人。
