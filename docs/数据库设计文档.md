## 一、集合总览

| 集合名称 | 说明 | 权限 | 负责人 |
|---------|------|------|--------|
| `users` | 用户信息 | 读取全部，修改本人 | 全员 |
| `announcements` | 公告/轮播图 | 读取全部，不可修改 | 成员A |
| `favorites` | 收藏记录 | 仅本人读写 | 成员A |
| `goods` | 二手商品 | 读取全部，修改本人 | 成员B |
| `lost_found` | 失物招领 | 读取全部，修改本人 | 成员C |
| `classrooms` | 教室信息 | 读取全部，不可修改 | 成员C |
| `reservations` | 自习预约 | 仅本人读写 | 成员C |

## 二、各集合字段定义

### 1. `users` — 用户信息

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| `_id` | String | 自动 | 系统自动生成 |
| `_openid` | String | 自动 | 微信自动填入，用户唯一标识 |
| `nickName` | String | ✅ | 用户昵称 |
| `avatarUrl` | String | ✅ | 头像图片地址 |
| `createTime` | Date | ✅ | 注册时间 |

**示例数据：**
```json
{
  "_id": "xxxx",
  "_openid": "o6zAJsxxxxx",
  "nickName": "张三",
  "avatarUrl": "cloud://xxx/avatar.jpg",
  "createTime": "2026-06-01T10:00:00.000Z"
}
```

---

### 2. `announcements` — 公告/轮播图

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| `_id` | String | 自动 | 系统自动生成 |
| `title` | String | ✅ | 公告标题 |
| `content` | String | ❌ | 公告正文内容 |
| `image` | String | ❌ | 轮播图图片地址 |
| `createTime` | Date | ✅ | 发布时间 |

**示例数据：**
```json
{
  "_id": "xxxx",
  "title": "校园运动会报名通知",
  "content": "2026年校园运动会将于6月15日举行...",
  "image": "cloud://xxx/banner1.jpg",
  "createTime": "2026-06-01T08:00:00.000Z"
}
```

---

### 3. `favorites` — 收藏记录

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| `_id` | String | 自动 | 系统自动生成 |
| `_openid` | String | 自动 | 收藏者标识 |
| `goodsId` | String | ✅ | 收藏的商品ID（对应 `goods` 集合的 `_id`） |
| `createTime` | Date | ✅ | 收藏时间 |

**示例数据：**
```json
{
  "_id": "xxxx",
  "_openid": "o6zAJsxxxxx",
  "goodsId": "goods_001",
  "createTime": "2026-06-02T14:30:00.000Z"
}
```

---

### 4. `goods` — 二手商品（成员B 核心集合）

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| `_id` | String | 自动 | 系统自动生成 |
| `_openid` | String | 自动 | 卖家标识 |
| `title` | String | ✅ | 商品标题 |
| `description` | String | ✅ | 商品描述 |
| `price` | Number | ✅ | 价格（单位：元） |
| `images` | Array\<String\> | ❌ | 商品图片地址列表，最多9张 |
| `category` | String | ✅ | 分类：`书籍` / `电子产品` / `生活用品` / `服饰` / `其他` |
| `status` | String | ✅ | 状态：`active`（在售）/ `sold`（已售出）/ `off`（已下架） |
| `contactInfo` | String | ❌ | 联系方式（手机号或微信号） |
| `createTime` | Date | ✅ | 发布时间 |

**示例数据：**
```json
{
  "_id": "goods_001",
  "_openid": "o6zAJsxxxxx",
  "title": "高等数学教材（第七版）",
  "description": "九成新，无笔记无划线，含课后习题答案",
  "price": 15,
  "images": ["cloud://xxx/img1.jpg", "cloud://xxx/img2.jpg"],
  "category": "书籍",
  "status": "active",
  "contactInfo": "13800138000",
  "createTime": "2026-06-01T12:00:00.000Z"
}
```

---

### 5. `lost_found` — 失物招领（成员C 核心集合）

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| `_id` | String | 自动 | 系统自动生成 |
| `_openid` | String | 自动 | 发布者标识 |
| `type` | String | ✅ | 类型：`lost`（我丢了东西）/ `found`（我捡到东西） |
| `title` | String | ✅ | 物品名称 |
| `description` | String | ✅ | 详细描述 |
| `images` | Array\<String\> | ❌ | 物品图片列表 |
| `category` | String | ✅ | 分类：`证件` / `电子产品` / `生活用品` / `其他` |
| `location` | String | ❌ | 丢失/捡到地点 |
| `contactInfo` | String | ❌ | 联系方式 |
| `status` | String | ✅ | 状态：`active`（进行中）/ `resolved`（已解决） |
| `createTime` | Date | ✅ | 发布时间 |

**示例数据：**
```json
{
  "_id": "lf_001",
  "_openid": "o6zAJsxxxxx",
  "type": "lost",
  "title": "校园卡",
  "description": "蓝色校园卡，姓名李四，昨天下午在图书馆丢的",
  "images": [],
  "category": "证件",
  "location": "图书馆三楼",
  "contactInfo": "微信：lisi123",
  "status": "active",
  "createTime": "2026-06-01T16:00:00.000Z"
}
```

---

### 6. `classrooms` — 教室信息（成员C）

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| `_id` | String | 自动 | 系统自动生成 |
| `building` | String | ✅ | 教学楼名称，如 `A栋`、`B栋` |
| `roomNo` | String | ✅ | 教室号，如 `301` |
| `capacity` | Number | ✅ | 座位数 |
| `status` | String | ✅ | 当前状态：`free`（空闲）/ `occupied`（占用中） |

**示例数据：**
```json
{
  "_id": "room_001",
  "building": "A栋",
  "roomNo": "301",
  "capacity": 60,
  "status": "free"
}
```

---

### 7. `reservations` — 自习预约（成员C）

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| `_id` | String | 自动 | 系统自动生成 |
| `_openid` | String | 自动 | 预约者标识 |
| `classroomId` | String | ✅ | 预约的教室ID（对应 `classrooms` 集合） |
| `date` | String | ✅ | 预约日期，格式 `2026-06-01` |
| `timeSlot` | String | ✅ | 时间段：`08:00-12:00` / `14:00-18:00` / `19:00-22:00` |
| `status` | String | ✅ | 状态：`active`（有效）/ `cancelled`（已取消） |
| `createTime` | Date | ✅ | 预约时间 |

**示例数据：**
```json
{
  "_id": "res_001",
  "_openid": "o6zAJsxxxxx",
  "classroomId": "room_001",
  "date": "2026-06-05",
  "timeSlot": "08:00-12:00",
  "status": "active",
  "createTime": "2026-06-01T20:00:00.000Z"
}
```

## 三、集合关系图

```
users (openid)
 ├──┬── favorites.goodsId  →  goods._id
 │  │
 │  └── reservations.classroomId  →  classrooms._id
 │
 ├── goods._openid  (用户发布的商品)
 │
 └── lost_found._openid  (用户发布的失物信息)
```

## 四、分类枚举值约定（全员统一使用）

### goods.category 商品分类
```
书籍 | 电子产品 | 生活用品 | 服饰 | 其他
```

### goods.status 商品状态
```
active（在售）| sold（已售出）| off（已下架）
```

### lost_found.type 失物招领类型
```
lost（我丢了东西）| found（我捡到东西）
```

### lost_found.category 物品分类
```
证件 | 电子产品 | 生活用品 | 其他
```

### lost_found.status 失物状态
```
active（进行中）| resolved（已解决）
```

### classrooms.status 教室状态
```
free（空闲）| occupied（占用中）
```

### timeSlot 时间段
```
08:00-12:00 | 14:00-18:00 | 19:00-22:00
```
