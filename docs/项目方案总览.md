## 二、项目背景与意义

### 2.1 背景

大学校园中，学生的日常需求分散在多个渠道：
- 课表查询需要登录教务系统
- 二手交易依赖 QQ群/微信群，信息杂乱
- 失物招领靠线下公告栏或朋友圈转发
- 自习室/空教室信息获取不便

学生需要一个统一、便捷的平台来满足这些日常需求。

### 2.2 意义

1. **学生层面**：一个入口解决多种校园需求，节省时间，提升校园生活体验
2. **学校层面**：促进校内资源循环利用（二手交易），降低管理成本
3. **技术层面**：实践微信小程序开发全流程，积累项目经验

## 三、核心功能模块

项目规划 **5个核心模块**：

| 序号 | 模块名称 | 核心功能描述 |
|------|---------|-------------|
| 1 | 🏠 **首页/资讯** | 校园公告轮播、快捷入口导航、热门推荐、信息聚合展示 |
| 2 | 📦 **二手交易** | 商品发布（图文）、分类浏览、关键词搜索、商品收藏、联系卖家 |
| 3 | 🎯 **失物招领** | 发布失物/招领信息、分类筛选、关键词搜索、状态标记（已认领/已找到） |
| 4 | 📚 **自习助手** | 空教室查询、自习室状态查看、自习室预约记录 |
| 5 | 👤 **个人中心** | 微信授权登录、我的发布、我的收藏、个人信息管理、联系客服 |

## 五、技术选型（已定稿）

| 层面 | 技术方案 | 说明 |
|------|---------|------|
| **前端框架** | 微信小程序原生框架 | 纯原生开发：WXML + WXSS + JS |
| **UI 组件库** | 无（纯原生 WXSS） | ✅ 已确认 — 手写样式，不引入第三方组件库 |
| **后端/云服务** | 微信云开发（CloudBase） | ✅ 已确认 — 云函数 + 云数据库 + 云存储，免服务器部署 |
| **数据库** | 云开发数据库（NoSQL） | 类 MongoDB，文档型存储 |
| **图片存储** | 云存储 | 商品图片、用户头像等 |
| **用户认证** | 微信授权登录（云开发免鉴权登录） | 获取用户 openid 和基本信息，代码量极简 |
| **版本管理** | Git + Gitee（码云） | ✅ 已确认 — 团队协作开发 |

## 六、数据库集合设计（初步）

| 集合名称 | 说明 | 主要字段 |
|---------|------|---------|
| `users` | 用户信息 | openid, nickName, avatarUrl, createTime |
| `goods` | 二手商品 | title, description, price, images, category, sellerId, status, createTime |
| `lost_found` | 失物招领 | type(失物/招领), title, description, images, category, contactInfo, status, createTime |
| `classrooms` | 教室信息 | building, roomNo, capacity, status |
| `reservations` | 自习预约 | userId, classroomId, date, timeSlot, createTime |
| `favorites` | 收藏记录 | userId, goodsId, createTime |
| `announcements` | 公告/资讯 | title, content, image, createTime |
