## 第一步：认识微信开发者工具界面

打开微信开发者工具后，窗口布局是这样的：

```
┌─────────────┬──────────────────────────────┐
│  左 侧     │                              │
│  文件树     │       中间：代码编辑区        │
│  (模拟器   │      （WXML/WXSS/JS）         │
│   上方)    │                              │
│            │                              │
│ cloudfunc/ │                              │
│ pages/     │                              │
│ ...        │                              │
├─────────────┴──────────────────────────────┤
│              底部：调试器                   │
│  ┌── Console ── Sources ── Network ── ... ┐│
│  │  > 这里就是控制台/调试器               ││
│  │  小程序运行日志、报错都在这里显示       ││
│  └────────────────────────────────────────┘│
└──────────────────────────────────────────────┘
```

| 区域 | 位置 | 作用 |
|------|------|------|
| **模拟器** | 窗口左侧上半部 | 预览小程序界面，可点击交互 |
| **文件树** | 窗口左侧下半部 | 项目文件和目录结构 |
| **代码编辑器** | 窗口中间 | 打开和编辑代码 |
| **调试器（Console）** | 窗口底部 | 查看日志、报错、网络请求 |

> 💡 如果底部调试器没有显示，点开发者工具顶部菜单栏 → **「调试器」**即可打开。

## 第二步：部署 login 云函数

### 2.1 找到 login 文件夹

在左侧文件树中，展开 `cloudfunctions` 目录，你会看到 `login` 文件夹：

```
cloudfunctions/
  └── login/          ← 就是这个！
       ├── index.js
       └── package.json
```

### 2.2 右键部署

用**鼠标右键**点击 `login` 文件夹，弹出菜单，点击：

> **「上传并部署：云端安装依赖」**

⚠️ **注意选第二个「云端安装依赖」，不要选第一个「上传」**。  
因为 `login/package.json` 里声明了 `wx-server-sdk` 依赖包，需要云端自动安装。

```
右键菜单：
┌─────────────────────────┐
│ 上传                     │  ← 不要选这个
│ 上传并部署：云端安装依赖  │  ← ✅ 选这个！
│ 下载云端代码             │
│ ...                      │
└─────────────────────────┘
```

### 2.3 等待完成

点完之后，底部调试器的 **Console** 选项卡会显示部署进度，看到这个就说明成功了：

```
✓ 云函数 login 上传成功
```

## 第三步：验证部署是否成功（3种方式）

### 方式一：在云开发控制台测试

1. 点击开发者工具**顶部工具栏**的 **「云开发」**按钮（图标是一朵云 ☁️）

   ```
   顶部工具栏：
   [云开发] [编译] [预览] [真机调试] ...
   ```

2. 在弹出的云开发控制台中，点击左侧菜单 **「云函数」**
3. 在右侧会看到 `login` 函数名以及右侧状态显示 **「部署成功」**
4. 点击 `login` 这一行，右侧展开详情标签页
5. 点击 **「测试」**标签页，再点击 **「调用」**按钮
6. 返回结果类似下面这样，拿到 `openid` 就说明正常：

   ```json
   {
     "openid": "o6zAJsxxxxxxxxxxxx",
     "appid": "wx4f16bf796bedaee0",
     "unionid": "oxxxxxxxxxxxxxxxx"
   }
   ```

### 方式二：看 Console 日志

1. 在开发者工具顶部工具栏点击 **「编译」**按钮（或按 `Ctrl + B`）
2. 立刻看底部 **Console**（控制台）选项卡
3. `app.js` 启动时会自动调用 login 云函数，如果一切正常，不会有红色报错
4. 如果有报错（比如 `errCode: -1`、`env not exists` 等），说明云函数没部署好或环境ID不对

### 方式三：在模拟器中操作验证

1. 编译运行小程序
2. 在模拟器中点击底部的 **「我的」** Tab
3. 点击 **「点击登录」**按钮
4. 如果弹出微信授权弹窗，授权后显示昵称头像，说明云函数正常工作
5. 如果弹 Toast 提示「网络异常，请重试」，说明云函数调用失败

## 第四步：插入测试数据

### 4.1 打开数据库控制台

点击开发者工具顶部工具栏的 **「云开发」**按钮 → 左侧菜单选择 **「数据库」**

```
云开发控制台界面：
┌────────────┬──────────────────────┐
│ 数据库  ◀  │  集合列表             │
│ 云函数     │  ├── announcements    │
│ 云存储     │  ├── classrooms       │
│ ...        │  ├── favorites        │
│            │  ├── goods            │
│            │  ├── lost_found       │
│            │  ├── reservations     │
│            │  └── users            │
└────────────┴──────────────────────┘
```

### 4.2 逐个集合插入数据

以 `announcements` 为例：

1. 点击左侧 **`announcements`** 集合名称
2. 点击右侧上方的 **「添加记录」**按钮
3. 在弹出的编辑器中，**删掉默认内容**，将下面的 JSON **原样粘贴**进去
4. 点击 **「确定」**保存
5. 重复以上步骤插入第2条、第3条

然后对 `goods`、`lost_found`、`classrooms` 做同样操作。

> 💡 如果「添加记录」按钮是灰色的，先点击集合名称让它高亮选中。

### users（用户数据）— 无需手动插

`users` 集合的数据由**小程序登录功能自动创建**。在模拟器中点击「我的」→「点击登录」→ 微信授权通过后，系统会自动向 `users` 集合写入一条用户记录。

## 第五步：预期效果

插入全部数据后，重新编译（Ctrl+B），首页应显示：

| 区域 | 预期效果 |
|------|---------|
| **轮播图** | 自动轮播 3 条公告（上线通知、运动会、图书馆） |
| **快捷入口** | 4 个图标：二手交易、失物招领、自习助手、更多服务 |
| **热门推荐** | 显示 10 条混合数据（5条商品 + 4条失物招领 + 1条不会重复的，按时间倒序） |

如果首页仍然空白：

1. 检查调试器 Console 有没有**红色报错**
2. 检查云开发环境 ID 是否正确（`app.js` 中 `env: 'cloudbase-d3glfg3zf894b18e'`）
3. 检查数据库集合名是否拼写正确（大小写敏感）
4. 检查数据权限设置：在云开发控制台 → 数据库 → 点击集合名 → 「权限设置」，确保权限为「所有用户可读」
