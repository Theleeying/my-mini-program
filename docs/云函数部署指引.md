# 云函数部署指引

> 📅 2026年5月31日

---

## 一、部署 login 云函数

### 步骤

1. **打开微信开发者工具**，确保项目已导入且左侧文件树可见。

2. 在左侧文件树中找到 `cloudfunctions/login/` 目录。

3. **右键 `login` 文件夹** → 选择 **「上传并部署：云端安装依赖」**。

   > ⚠️ 注意：选「云端安装依赖」而不是「上传」，因为 `login` 的 `package.json` 里声明了 `wx-server-sdk` 依赖。云端安装会自动下载依赖包。

4. 等待上传完成，控制台会显示：
   ```
   ✓ 云函数 login 上传成功
   ```

5. **验证是否部署成功：**
   - 方式一：在微信开发者工具中，选择「云开发」→「云函数」，如果看到 `login` 且状态为「部署成功」，则说明部署完成。
   - 方式二：刷新小程序首页，打开调试器 Console，如果看到用户 openid 正常打印（或有网络请求返回 openid），则说明云函数调用成功。
   - 方式三：在云开发控制台 → 云函数 → 点击 `login` → 点击「测试」，如果返回 JSON 包含 openid，则验证通过。

### 注意事项

- 如果提示「未选择云环境」，请先在微信开发者工具顶部点击「云开发」按钮，确认当前环境为 `cloudbase-d3glfg3zf894b18e`。
- 如果部署后调用失败，检查 `app.js` 中的 `env` 字段是否填写正确。

## 二、插入测试数据

### 方式一：在云开发控制台手动添加（推荐）

1. 打开微信开发者工具 → 顶部「云开发」按钮 → 进入云开发控制台
2. 左侧菜单选择「数据库」
3. 选择对应集合 → 点击「添加记录」
4. 将下方对应的 JSON 数据粘贴进去 → 点击「保存」

> 也可以直接访问：https://tcb.cloud.tencent.com/dev#/db/doc

### 方式二：通过云函数批量导入

可以在 `cloudfunctions/` 下新建一个 `initData` 云函数，一次性批量写入所有测试数据。

### 4. `users` 集合 — 插入 1 条测试用户

```json
{
  "nickName": "测试用户张三",
  "avatarUrl": "",
  "createTime": "2026-06-01T10:00:00.000Z"
}
```

> ⚠️ 这条数据不带 `_openid`（云开发会自动填入），如果通过控制台手动添加，`_openid` 字段会自动关联当前登录的微信用户。建议**直接通过小程序内的登录功能来创建用户记录**，这样 `_openid` 会自动匹配。
